Ayuda — Research Lab

Proyectos

Desde la pantalla de inicio podés crear o seleccionar un proyecto de investigación. Cada proyecto tiene miembros con rol PI (investigador principal) o Colaborador. Solo el PI puede editar o archivar el proyecto e invitar nuevos miembros.

Bitácora

Registro inmutable de eventos del proyecto. Usá tipos para categorizar cada entrada:

  • Progreso — avances de trabajo
  • Modificación — cambios de alcance o diseño
  • Decisión — resoluciones del equipo
  • Hito — logros alcanzados
  • Nota — observaciones generales

Las entradas aceptan formato Markdown (negrita, cursiva, listas, código).

Hipótesis

Gestioná las hipótesis de investigación con estados: Pendiente → En progreso → Validada / Rechazada / En espera. La prioridad va de 1 (crítica) a 5 (baja).

Hitos y Requerimientos

Creá hitos con fecha límite. Cada hito puede tener requerimientos (tareas), asignables a miembros del equipo con estados: pendiente, en progreso, listo, bloqueado.

Notas

Editor Markdown con vista previa. Las notas se organizan en carpetas virtuales. Podés usar [[Título de nota]] para enlazar automáticamente nodos en el Knowledge Graph.

Bibliografía

Biblioteca de referencias bibliográficas (papers, libros, datasets, estándares, web). Podés agregar DOI, URL, resumen y notas propias. Las citas se formatean automáticamente en estilo APA.

Knowledge Graph

Visualización de relaciones entre nodos del proyecto (hipótesis, notas, hitos, referencias). Las conexiones pueden ser:

  • Explícitas — creadas manualmente con el botón "+ Relación" o desde el panel del nodo
  • Automáticas — detectadas desde [[wikilinks]] en notas e hipótesis (línea punteada)

Podés filtrar por tipo, arrastrar nodos, hacer zoom con la rueda del mouse y hacer clic en un nodo para ver sus conexiones en el panel lateral.

Navegación por menú
  • Esc — cerrar menú abierto o modal
  • Clic fuera del menú — cierra los menús desplegables
Notas — editor
  • Ctrl + S — guardar nota (también guardado automático)
  • El guardado automático ocurre 1.2 s después del último cambio
Formato Markdown (Bitácora y Notas)
  • **texto** → negrita
  • _texto_ → cursiva
  • # Título → encabezado
  • - ítem → lista
  • `código` → código en línea
  • [[Título]] → wikilink a otro nodo del proyecto
Knowledge Graph
  • Rueda del mouse — zoom in/out
  • Arrastrar fondo — pan (mover la vista)
  • Arrastrar nodo — reposicionar nodo
  • Clic en nodo — abrir panel de conexiones
  • Botón ↺ — recargar grafo
¿Qué es la GitHub App del proyecto?

Cada proyecto puede conectar su propio repositorio GitHub a través de una GitHub App. Esto permite que cualquier colaborador proponga notas, hipótesis, bitácora y más como Pull Requests, sin necesitar una cuenta de GitHub personal. El PI del proyecto la configura una sola vez.

Parte 1 — Crear la GitHub App en GitHub

Hacé estos pasos desde tu cuenta de GitHub (o desde la cuenta de la organización):

  1. Ingresá a GitHub → Settings → Developer settings → GitHub Apps → New GitHub App
  2. Completá los campos obligatorios:
    GitHub App name: un nombre único (ej. Mi-Lab-Investigacion)
    Homepage URL: la URL del Lab (ej. https://lab.aural-syncro.com.ar)
    Webhook → Active: desmarcar (no se necesita)
  3. En Repository permissions configurá solo lo necesario:
    • Contents → Read & write
    • Pull requests → Read & write
    • Metadata → Read-only (obligatorio)
  4. En Where can this GitHub App be installed? elegí "Any account" para que otros PI puedan instalarla también.
  5. Hacé clic en Create GitHub App. Vas a ver el App ID (número) arriba del nombre — anotalo.
  6. Bajá hasta Private keys → Generate a private key. Se descarga un archivo .pem — guardalo de forma segura.
  7. Convertí la clave a base64 en una terminal: openssl base64 -in tu-app-private-key.pem | tr -d '\n' Copiá toda la cadena resultante (comienza con LS0tLS1...).
  8. Instalá la App en tu cuenta u organización: andá a https://github.com/apps/{nombre-de-tu-app} → Install → elegí el repositorio específico o todos → Install.
  9. Después de instalar, GitHub te redirige a una URL del tipo:
    https://github.com/settings/installations/12345678 El número al final es el Installation ID — anotalo.
Si el repositorio todavía no existe podés dejarlo para que el Lab lo cree automáticamente al conectar.
Parte 2 — Conectar en el Lab

Solo el PI del proyecto puede hacer este paso:

  1. Abrí el proyecto → menú Conocimiento → Repositorio GitHub.
  2. Hacé clic en Conectar repositorio → y completá el formulario:
    • App ID — el número anotado en el paso 5
    • Clave privada (base64) — la cadena del paso 7
    • Installation ID — el número del paso 9
    • Owner — tu usuario u organización de GitHub
    • Repositorio — nombre del repo existente, o tildá "Crear automáticamente"
  3. Hacé clic en Conectar App →. Si todo está correcto el Lab valida las credenciales en tiempo real y el repositorio queda enlazado.
Las credenciales (App ID y clave privada) se almacenan cifradas en la base de datos del proyecto. Ningún colaborador puede verlas ni exportarlas.
Proponer contenido al repositorio

Con el repo conectado, cualquier colaborador puede ir a Repositorio GitHub → Proponer contenido, elegir el tipo (nota, hipótesis, bitácora, hito, referencia), seleccionar el ítem y escribir el mensaje del commit. El Lab crea una branch y abre el PR automáticamente, con el footer de autoría del colaborador.

El PI puede mergear los PRs directamente desde el Lab sin abrir GitHub.

Aural-Syncro · Research Lab

Plataforma de gestión de investigación científica desarrollada por el equipo de Aural-Syncro. Permite documentar, organizar y relacionar el conocimiento generado durante proyectos de investigación.

Módulos implementados
  • Autenticación y control de miembros por roles (PI / Colaborador / Observador)
  • Bitácora de investigación con formato Markdown
  • Gestión de hipótesis con prioridades y estados
  • Hitos y requerimientos con asignaciones
  • Notas tipo Obsidian con carpetas virtuales y guardado automático
  • Biblioteca bibliográfica con citas APA
  • Knowledge Graph con D3.js — relaciones explícitas y auto-detectadas
  • Repositorio GitHub — proponer contenido como PR, listar y mergear PRs (DT-RL-009)
